/v2/client/{clientId}/registrant

METHOD: GET

Returns a listing of registrants for a particular client. There is a limit of 100 registrants per page.

Notes:

  • If double opt-in verification was enabled at the time of registration for an event, and a user has not completed the verification process, the user will not be included in any endpoint responses.
  • subaccounts parameter conditions:
    • Value can be 1 or more child client IDs (multiple values will need to be delimited by a comma)
    • Child client IDs must be under the {clientId} of the request
    • If subaccounts is provided and excludeSubaccounts=Y, following error message is returned: "You have to set excludesubaccounts='N' in order to filter by subaccounts.".
    • If subaccounts includes multiple values, and at least one value is not valid, an error message is returned.
    • When subaccounts is provided, the response will only return the data of the child client IDs included in the parameter.
  • dateFilterMode parameter options (not case-sensitive):
    • If the parameter is not included or no value is supplied, startDate and endDate will work in the manner they usually do.
    • "lastActivity" is the only supported value. When passed, the response will include only those registrants who have had activity between (and including) the provided startDate and endDate. Registrant lastActivity value is updated with new modified date whenever one of the following API endpoints gets invoked to modify the registration information:
      • REST Registration
      • Update Registrant (client-level)
      • Forget Registrant (client-level)
      • Forget All Registrant (client-level)
      • Forget All Registrant (event-level)
