/v2/client/{clientId}/attendee/{email}/allevents

METHOD: GET

Returns a listing of all attendance information for a specific email address. There is a limit of 100 registrants per page.

Parameters:

Name Located in Required Description Default Schema Example
clientId path yes The client id - string 15448
email path yes The email to lookup - string john.doe@on24.com
accessTokenKey header yes client access token key - string 63129107067A520E6B7DE18D5E1CB2BF
accessTokenSecret header yes client access token secret - string 072639740ac239333e3708a1623b9de3590d972e6b1060071b726ecfd9b216001b45391717d9fbeb780438
excludeSubaccounts query no if set to "y", sub-accounts will be excluded N string Y
subaccounts query no Comma separated list of child client ids - string 22921, 32295
pageOffset query no Selects the page to retrieve 0 integer 1
itemsPerPage query no Number of items to be retrieved per page 100 integer 25

 

Notes:

  • subaccounts parameter conditions:
    • Value can be 1 or more child client IDs (multiple values will need to be delimited by a comma).
    • Child client IDs must be under the {clientId} of the request.
    • If subaccounts is provided and excludeSubaccounts is "Y", the following error message is returned: "You have to set excludesubaccounts='N' in order to filter by subaccounts."
    • If subaccounts includes multiple values and at least one value is not valid, an error message is returned.
    • When subaccounts is provided, the response will only return data for the child client ids included in the parameter

 

Response:

Content-Type: application/json

Status Code Reason
200 Return list of attendees
401 AccessTokenKey and AccessTokenSecret are wrong or invalid.
403 You do not have permission to perform the requested operation.
403 You have reached the maximum number of calls for this api.
404 The attendant {email} doesn't exist.
404 The client {clientid} doesn't exist.
