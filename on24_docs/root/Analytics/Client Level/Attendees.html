/v2/client/{clientId}/attendee

METHOD: GET

Returns a listing of attendees for a particular client. There is a limit of 100 attendees per page.

Notes:

  • subaccounts parameter conditions:
    • Value can be 1 or more child client ids (multiple values will need to be delimited by a comma).
    • Child client IDs must be under the {clientId} of the request
    • If subaccounts is provided and excludeSubaccounts is "Y", following error message is returned: "You have to set excludesubaccounts='N' in order to filter by subaccounts.".
    • If subaccounts includes multiple values and at least one value is not valid, an error message is returned.
    • When subaccounts is provided, the response will only return data for the child client ids included in the parameter

  • dateFilterMode parameter conditions (not case-sensitive):

    • If the parameter is not included or no value is supplied, startDate and endDate will work in the manner they usually do.

    • "lastActivity" is the only supported value. When passed, the response will include only those attendees who have had activity between (and including) the provided startDate and endDate. Attendee lastActivity value is updated with new modified date whenever one of the following actions is performed by an attendee:

      • Starting new session

      • Refreshing browser during webcast presentation

      • Submitting poll answers

  • realtimeonly parameter conditions (when "Y" is passed):

    • Properties that rely on the datawarehouse will not be included in the response, including:

      • engagementscore

      • firstliveactivity

      • lastliveactivity

      • firstarchiveactivity

      • lastarchiveactivity

      • askedquestions

      • resourcesdownloaded

      • answeredpolls

      • answeredsurveys

      • cumulativeliveminutes

      • cumulativearchiveminutes

      • Any activity / updates that have occurred in the last 15 minutes will be available in the response.
