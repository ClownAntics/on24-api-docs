/v2/client/{clientId}/userquestions

METHOD: GET

Returns a listing of user questions for a particular client.

Parameters:

Name Located in Required Description Default Schema Example
clientId path yes The client id - string 15448
excludesubaccounts query no if set to "Y", sub-accounts will be excluded N string Y
emailFilter query no Limit the questions returned to only those from specific domains - string microsoft.com|intel.com|ba.com
startDate query no Limit the results to those occurring after this date/time 5 minutes ago only ISO format: YYYY-MM-DDThh:mm:ssXXX 2012-05-19T12:00:00-05:00
endDate query no limit the results to those occurring before this date/time now only ISO format: YYYY-MM-DDThh:mm:ssXXX 2012-05-19T12:00:00-05:00
accessTokenKey header yes client access token key - string 63129107067A520E6B7DE18D5E1CB2BF
accessTokenSecret header yes client access token secret - string 072639740ac239333e3708a1623b9de3590d972e6b1060071b726ecfd9b216001b45391717d9fbeb780438

 

Notes:

  • Real-time API endpoint that returns the info for the last 5 minutes
  • The maximum duration for the startDate and endDate interval is 1 hour.

 

Response:

Content-Type: application/json, application/xml, text/plain

Status Code Reason
200 Return list of user questions
400 the startDate and endDate are more than 1 hour apart.
401 AccessTokenKey and AccessTokenSecret are wrong or invalid.
403 You do not have permission to perform the requested operation.
403 You have reached the maximum number of calls for this api.
404 The client {clientId} doesn't exist.
