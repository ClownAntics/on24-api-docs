/v2/client/{clientId}/event/{eventid}/surveyquestions

METHOD: POST

Creates one or several survey questions and their answers for a particular webinar.

Parameters:

Name Located in Required Description Default Schema Example
clientId path yes The client id - string 18091
eventId path yes The event id - string 13658130
accessTokenKey header yes client access token key - string 63129107067A520E6B7DE18D5E1CB2BF
accessTokenSecret header yes client access token secret - string 072639740ac239333e3708a1623b9de3590d972e6b1060071b726ecfd9b216001b45391717d9fbeb780438
content-type header yes It must be either “application/x-www-form-urlencoded" or "application/json" depending on the preferred way to send the metadata object: as a formData parameter or as the request body. - string "application/x-www-form-urlencoded"
metadata

formData

or

requestBody

 yes JSON object containing all the survey questions data. It can be sent as a formData parameter using Content-Type = "application/x-www-form-urlencoded" or as the request body using Content-Type = "application/json". - JSON {
    "surveyquestions": [
        {
            "questiontype": "multioption",
            "required": "Y",
            "question": "What industry are you in?",
            "surveyanswers": [
                "Technology",
                "HealthCare",
                "Finance",
                "Retail",
                "Food & Beverage"
            ]
        },
        {
            "questiontype": "singleoption",
            "required": "N",
            "question": "Would you like a 1:1 session with an expert?",
            "surveyanswers": [
                "Yes",
                "No"
            ]
        },
        {
            "questiontype": "singletext",
            "required": "N",
            "question": "Describe yourself"
        }
    ]
}

 

Notes:

  • A Take Survey widget must already exist in the webinar. If a Take Survey widget does not exist, a 404 response will be returned.

 

Response:

Content-Type: application/json

Status Code Reason
200 Questions created.
400 Invalid metadata.
401 AccessTokenKey and AccessTokenSecret are wrong or invalid.
401 You do not have permission to perform the requested operation.
403 You do not have permission to perform the requested operation.
403 You have reached the maximum number of calls for this api.
403 The client id {clientId} is inactive.
403 The event {eventId} is inactive.
404 The client {clientId} doesn't exist. 
404 The event {eventId} doesn't exist.
404 The Take Survey widget is not available for this event.
