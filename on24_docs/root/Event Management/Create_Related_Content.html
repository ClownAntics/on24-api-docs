/v2/client/{clientId}/event/{eventid}/relatedcontent

METHOD: POST

This endpoint will upload files and links to a Webinar's Related Content widget.

Parameters:

Name Located in Required Description Default Schema Example
clientId path yes The client id - string 15448
eventId path yes The event id - string 65423
accessTokenKey header yes The client access token key - string 63129107067A520E6B7DE18D5E1CB2BF
accessTokenSecret header yes The client access token secret - string

Content-Type header yes Must be "multipart/form-data". - string multipart/form-data
matchname formData yes Used to find the name of the existing Related Content widget - string

Related Content 1
name formData yes Display name of the resource being uploaded - string ON24 Website
type formData yes Determines whether the resource being uploaded is either a file or a URL. Accepted values are: "Resource" for uploading a file and "URL" for uploading a URL. These values are case-sensitive. - string URL
file formData no The file to be uploaded if type is "Resource". This parameter becomes mandatory when type is "Resource" and must not be sent at all when type is "URL". - file whitepaper.pdf
url formData no URL for the resource if type is "URL". This parameter becomes mandatory when type is "URL" and must not be sent at all when type is "Resource". - string https://www.on24.com
