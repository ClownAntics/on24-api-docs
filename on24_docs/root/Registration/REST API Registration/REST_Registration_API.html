/v2/client/{clientId}/event/{eventId}/registrant

METHOD: POST

Creates a registrant.

Soft Delete

Perform a soft delete operation on a particular Registrant by sending the 'isDeleted' form parameter with the value of (Y or N) to update the deleted state of such Registrant. This value can be sent along with the other form parameters listed in the table above.

Deleting/Undeleting a registrant is not allowed after the event has started.
Name Located In Required Description Default Schema Example
isDeleted formData yes Performs an update on the isDeleted property of a Registrant - string Y
email formData yes The email of the registrant who will be updated - string john.doe@domain.com

 

Validation

  • If a field is configured as "required" on the registration page, you must pass in a valid value, or the request will not be successful unless you are using the honorrequired as "N" parameter.
  • The honorrequired parameter is supported with possible values of Y or N. If Y is passed (or no value or no parameter is passed), the endpoint will behave as it currently does. If N is passed:
    • Fields configured as required will be optional (including email)
    • If a drop-down field is passed, the value must adhere to configured options unless you are using the honorvalidation as "N".

 

Response

Content-Type: application/json, application/xml, text/plain

404 The event {eventId} doesn't exist.
412 You're trying to soft delete the registrant you're creating.
