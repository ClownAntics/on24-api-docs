Seamless Registration Virtual Conference

Seamless Registration is a service for pushing registrant data to Virtual Conferences. This service supports both the GET and POST HTTP methods.

 

Registration API URL:

https://vshow.on24.com/vshow/<showCode>/register

 

Required Parameters

email User email address
password User chosen password (if applicable)
apiKey Preview password
apiMode If parameter provided, value always needs to be “Y”
target This URL parameter overrides the show lobby URL to redirect after successful registration. Only use if “apiMode” parameter is not passed or equal to “N”

First Time Registrants

 

If the system receives a registration request for a user whose information is not available in the data store (new registrant), the following steps are taken:

 

  1. Register the new user with the provided information.

  2. On successful registration, send out a confirmation email.

  3. Creation an “auth” cookie which will be valid for 5 minutes. The XML response will contain the auth token value as shown below.

  4. If the registration information includes tracking information for regPageId or partnerref, store this tracking information for the newly created user.

Successful Registration Response

<?xml version=\"1.0\" encoding=\"UTF-8\"?>

<response>

<status>ok</status>

<token>nON2KJA2kPb0Ixr0e0J71YWdKIm1Jmx%2FTYY6GKLVBKs%3D</token>

</response>

 

If using a server POST, then the token needs to be passed as a request paramter appended to the show URLor lobby URL using the below format.

https://vshow.on24.com/vshow/<showCode>?auth=<tokenValue>

 

Existing Users

If the system receives a registration request for a user and based on the email provided, it determines the user is already a valid registrant, the following steps are taken:

  1. If the user has already registered with the same regPageId, his/her next attempts will be ignored by the tracking system and no confirmation email will be sent.

  2. If the user has already registered but the new request contains a new regPageId, a confirmation email will be sent from the new registration page.

  3. Send a response that the user is already registered along with the auth cookie creation and the auth token value.

  4. If the registration information includes tracking information for regPageId or partnerref, store this tracking information for the newly created user.

User Already Registered

<?xml version=\"1.0\" encoding=\"UTF-8\"?>

<response>

<status>USER_ALREADY_REGISTERED</status>

<token>nON2KJA2kPb0Ixr0e0J71aPpWQKiqcZmyIlLeLEKPtI%3D</token>

</response>

 

Registration Confirmation Email

A registration confirmation email is sent out for every valid registration attempt per regPageId. If a registration value already exists for a user with a given regPageId, then no confirmation email will be sent out. If the registration information does not contain regPageId parameter, the system will use the default registration page.

 

Auto Login Token Generation API

This API is used to generate the token which will login the user automatically without being asked for the credentials again. This API creates the “auth” cookie as well as returns the token in the response. This token would be valid for approximately 5 minutes.

 

API URL:

https://vshow.on24.com/vshow/<showCode>/token/generate

 

Required Parameters

email

User email; needs to be unique for the show

password

User chosen password OR apiKey (show preview password)

apiMode

Y/N (If parameter not provided, default value = N)

target

This URL parameter overrides the Lobby URL to redirect on successful registration. Required if apiMode = N.

 

Other supported parameters

 

errorUrl

escaped URL; applicable when apiMode=N; optional; redirected to errorUrl on failed login with an additional request parameter appended to errorUrl called “errorCode” Eg. errorCode=USER_NOT_REGISTERED or errorCode=INVALID_USER_PASSWORD.

 

Successful Token Generation Response

<?xml version=\"1.0\" encoding=\"UTF-8\"?>

<response>

<token>nON2KJA2kPb0Ixr0e0J71aPpWQKiqcZmyIlLeLEKPtI%3D</token>

</response>
