Seamless Login Engagement Hub and Target

ON24 Engagement Hub and Target supports a Seamless Login. This allows access for existing registrants by passing through the email address or an external event user code as a query parameter for a redirect URL.

 

Sending Login Data

A script can be used with either an HTTP GET or HTTP POST method. The unique Engagement Hub ID (eventId) and Audience Key (key) parameters must be included regardless of the method you use to send the login data.

 

HTTP GET Method

The GET method relies on JavaScript redirection, which requires a web browser. This method will fail if used for server-to-server requests.

  • If any of the parameters include the SPACE character, it must be defined using URL encoding i.e. " ", (eg, &country=United States). Some special characters embedded within the field values, like "=", "&" etc., should also always be encoded.
  • It is recommended that you add the deletecookie=true parameter to avoid issues with users regsitering for the same event on one machine.
  • Example with email in the URL: https://gateway.on24.com/wcc/login?eventId=XXXXXX&sessionid=1&key=XXXXXXXXXXXXXXXXXXXXXXXX&email=john.smith@abc.com&targetUrl=
  • Example with external event user code in the URL: https://gateway.on24.com/wcc/login?eventId=XXXXXX&experienceId=XXXXXX&sessionid=1&key=XXXXXXXXXXXXXXXXXXXXXXXX&extEventUserCd=12345&targetUrl=

 

HTTP POST Method

  • UTF input isn’t necessary if sending UTF-8, which is the default. If you’re sending characters using a different encoding, change the hidden value to the proper character set. For example, ISO-8859-1.
  • Example request:

<form name="myform" action="https://gateway.on24.com/wcc/login" method="POST">

  <input type="hidden" name="eventId" value="XXXX">

  <input type="hidden" name="sessionid" value="1">

  <input type="hidden" name="key" value="XXXXXXXXXXXXXXXXXXX">

  <input type="text" name="email" value="john.smith@abc.com">

             <input type="hidden" name="targetUrl" value="">

  <input type="submit" value="Submit Form"/>

</form>

email or extEventUserCd

targetUrl

No

Case-1: If 'targetUrl' parameter is not present
Output: User will not be redirected to any page after login but API will return the JSON EH/Target event user object1

Case-2: If 'targetUrl' parameter is present but no value is being passed
Output: User will be redirected to Engagement Hub or Experience page

Case-3: If 'targetUrl' parameter is present with value being passed
Output: User will be redirected to the page specified in the parameter

 

Responses

The Seamless Login API provides error messages/codes when the required parameters are not being passed correctly. See below examples:

  • {"error":"ENGAGEMENT_HUB_NOT_FOUND","message":"Enagagement Hub doesn't exist"}

  • {"error":"INVALID_EVENT_ID_AND_KEY","message":"eventId/gatewayId and key doesn't match"}

  • {"error":"EMAIL_OR_EXTERNAL_EVENT_USER_CD_NOT_PASSED","message":"Email address/extEventUserCd is empty or not valid"}

  • {"error":"EH_USER_NOT_FOUND","message":"Enagagement Hub user not found"}

  • {"error":"TARGET_USER_NOT_FOUND","message":"Target user not found"}
