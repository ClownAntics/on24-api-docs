Unified Seamless Registration

Unified Seamless Registration is a service for pushing registrant data to Elite, Engagement Hub, Target Experience, Go Live, Forums and Virtual Conference. This service supports both the GET and POST HTTP methods.

HTTP GET
The GET method requires URL query parameters. Sample request: 

https://seamlessreg.on24.com/?eventid=1234&key=A95229314569C6039E5F9D4AE3EFD980&firstname=john&lastname=Doe&company=ON24&email=john.doe@on24.com

 

HTTP POST

The POST method requires parameters as part of the request body, and uses the header 'Content-Type': 'application/x-www-form-urlencoded'. Sample request:

<form action="https://seamlessreg.on24.com/" method="POST">
  <input type="hidden" name="eventid" value="5001937">
  <input type="hidden" name="key" value="A95229314569C8839E5F9D4AE3EFD980">
  <input type="text" name="firstname" value="john">
  <input type="text" name="lastname" value="smith">
  <input type="text" name="email" value="john.poiuyt@abc.com">
  <input type="text" name="company" value="ABC, Inc.">
  <input type="submit" value="Submit Form"/> 
</form>

Parameter Notes

  • key
    • For Elite and Forums, the key can be pulled from the Audience URL: https://event.on24.com/wcc/r/eventid/key

  • For Engagement Hub, Target and Go Live, pull the key from the Seamless Registration URL.

  • For VC, the key will be the Preview Password.

  • workphone and homephone validation:
    • Contain at least 7 digits
    • Non-numeric characters are removed
    • Phone numbers with 7 digits are formatted as ###-#### in the response
    • Phone numbers with 10 digits are formatted as (###)-###-#### in the response
    • Max allowed length: homephone=1000|workphone=1000|fax=1000 (if length is greater than 1000 it is truncated
  • custom1 - custom10
    • For webinars, custom1 - custom10 corresponds to std1 - std10.

 

Redirection Use Cases

The targeturl parameter is optional. Below is the behavior of both the GET and POST methods, with or without the parameter. Depending on the configuration, the response can be JSON, a redirect to the event, or a redirect to an external page.

 

Example EHub GET with targeturl parameter that would redirect to EHub: https://seamlessreg.on24.com/?eventid=1234&key=A95229314569C6039E5F9D4AE3EFD980&firstname=john&lastname=Doe&company=ON24&email=john.doe@on24.com&targeturl=

Virtual Conference Specific Configurations

1. VC requires one extra parameter to determine if redirection or JSON response is performed.

  • apimode
    • "Y" or "N"
    • If the parameter is not sent, it will redirect
    • If the parameter is 'N', it will redirect
    • If the parameter is 'Y', it will respond with JSON
